# 객체지향 프로그래밍
> Object-oriented programming (OOP)

프로그램을 객체 간의 상호작용으로 나타내는 프로그래밍 방식  

특징
- 높은 재사용성 (상속)
- 유지보수 용이 (객체 관계, 모듈화)
- 높은 보안성 및 신뢰성 (캡슐화, 중복 제거)

<br>

# SOLID 원칙
### SRP 단일 책임 원칙
> Single Responsibility Principle

클래스의 목적을 명확하게 하기  
너무 많은 일을 수행하지 않도록 하기  

<br>

### OCP 개방 폐쇄 원칙
> Open-Closed Principle

기능 추가를 고려하여 수정보다는 확장 가능한 코드로 작성하기  
추상화 및 상속을 잘 활용하기  

<br>

### LSP 리스코프 치환 원칙
> Liskov Substitution Principle

자식 클래스 객체가 부모 클래스의 기능을 정상적으로 수행하는지 확인하기  

<br>

### ISP 인터페이스 분리 원칙
> Interface Segregation Principle

클라이언트에게 필요없는 인터페이스 전달하지 않기  
인터페이스 분리를 잘 하기

<br>

### DIP 의존성 역전 원칙
> Dependency Inversion Principle

상위 클래스가 하위 클래스에 휘둘리지 않도록 하기  
하위 클래스 직접 참조 보다 추상 클래스 참조하기
